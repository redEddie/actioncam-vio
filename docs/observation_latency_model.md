# 관측 지연·지터 노이즈 모델 (VLA 학습용)

VLA 롤아웃에서 정책이 보는 이미지는 "지금"이 아니라 **지연 L만큼 과거**의
세계이고, L은 프레임마다 흔들린다(지터). action chunking이 지연의 영향을
완화하지만, **학습 시 같은 통계의 지연/지터를 주입**하면(sim2real 노이즈)
train-rollout 격차를 더 줄일 수 있다. 이 문서는 실측값과 그것을 재현하는
노이즈 모델을 기록한다 (측정: `gopro_vio/latency_test.py`, 2026-07-07).

## 1. 실측값 (glass-to-host, 모니터 응답 포함)

| 경로 | 조건 | median | p10 | p90 | 지터 폭(p10–p90) |
|---|---|---|---|---|---|
| **Ace Pro 2 UVC** | 1080p30 MJPG | **163 ms** | 145 | 185 | 40 ms |
| **HERO7 + MacroSilicon(MBF) 동글** | 720p60 MJPG | **186 ms** | 131 | 227 | 96 ms |

같은 방법(패턴 매 프레임 갱신, 보정 없음)으로 측정한 직접 비교 수치.
HERO7 동글 경로 추가 특성: 1080p30은 인접 프레임 블렌딩(이중상),
1080p60은 프레임 손상 → 720p60만 유효.

## 2. 측정치 → 카메라 파이프라인 지연 (보정)

측정값에는 실제 롤아웃에 없는 두 성분이 포함된다:
모니터 응답(~15 ms)과 패턴 나이(표시 루프 주기의 절반: Ace 측정 시 ~16 ms,
GoPro 측정 시 ~8 ms). 이를 빼면:

| 경로 | 롤아웃 지연 상수 μ | 지터 표준편차 σ* |
|---|---|---|
| Ace Pro 2 UVC | **≈132 ms** | ≈13 ms |
| HERO7 + MBF 동글 | **≈163 ms** | ≈35 ms |

*p10–p90 폭을 정규분포로 환산(폭≈2.56σ), Ace는 패턴 나이 분산 제거 포함.

## 3. 노이즈 모델

프레임별 관측 지연을 다음으로 샘플링한다:

```
L_t = μ + ε_t,   ε_t ~ N(0, σ²),   L_t를 [μ-3σ, μ+3σ]로 클립
```

| 경로 | 모델 |
|---|---|
| Ace Pro 2 | L ~ N(132, 13²) ms |
| HERO7+MBF | L ~ N(163, 35²) ms |

30 fps 학습 데이터에서는 프레임 인덱스 시프트로 구현된다
(1프레임 = 33.4 ms):

```python
import numpy as np

def delayed_frame_index(t_idx, fps=30.0, mu_ms=132.0, sigma_ms=13.0,
                        rng=np.random):
    """학습 샘플의 관측으로 '몇 프레임 전' 이미지를 쓸지 샘플링.

    action label은 t_idx 시점 그대로 두고, 이미지 관측만 과거로 민다.
    """
    L = np.clip(rng.normal(mu_ms, sigma_ms), mu_ms - 3*sigma_ms,
                mu_ms + 3*sigma_ms)
    return max(0, t_idx - int(round(L * fps / 1000.0)))
```

- Ace(132±13 ms): 시프트 4프레임 ± 0~1프레임 — 지터가 프레임 양자화보다
  작아 사실상 **상수 지연 4프레임**으로 근사해도 무방.
- HERO7+MBF(163±35 ms): 시프트 5프레임 ± 1프레임 — 지터 주입이 의미 있음.
- 에피소드 단위 상수 성분의 느린 변동(온도/부하)은 미측정 — 에피소드마다
  μ 자체를 ±10 ms 재샘플링하는 계층적 노이즈로 보수적으로 커버 가능.

## 4. 함께 주입할 만한 경로별 아티팩트 (선택)

| 아티팩트 | 대상 경로 | 증강 구현 |
|---|---|---|
| 프레임 블렌딩 | HERO7+MBF를 1080p30으로 쓸 경우 | `I' = 0.5·I_t + 0.5·I_{t-1}` (확률적 적용) |
| MJPEG 압축 | 두 경로 공통 | JPEG 재인코딩 품질 60~90 랜덤 |
| 기하 정합 | 두 경로 공통 | 지연 이전에 `gopro_vio.remap`으로 롤아웃 기하 변환 먼저 적용 |

적용 순서: **remap(기하) → 지연 시프트(시간) → 블렌딩/압축(화질)**.

## 5. 하드웨어가 바뀌면

캡처보드(Elgato HD60 X 등)나 카메라 교체 시 μ, σ만 재측정해 갱신:

```bash
python -m gopro_vio.latency_test <장치 by-id 경로> --size 1280x720 --fps 60 \
    --seconds 30 --setup
```

측정 원리와 주의(동글 블렌딩 시 --hold-ms, 뷰어 버퍼링과 혼동 금지)는
카메라 페이지의 지연 항목과 `latency_test.py` docstring 참조.
